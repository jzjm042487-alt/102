# 路径甲：两阶段解耦 + 对偶反馈回炉（工程务实版）

> 状态：设计草案 / 备选方案（工程复杂度低于路径乙）
> 目标读者：求解器开发者
> 关联文档：`route-b-rcg-design.md`（路径乙，理论最优）、`花样最少化文献导读.md`、`cutting-welding-best-practices.md`
> 定位：路径乙（R&CG）的**工程降级版**。当 R&CG 实现成本过高、或需要快速上线时，用本方案。它牺牲了严格收敛保证，换取实现简单、可直接复用现有 `solver.py`。

---

## 0. 一句话概括

**保持两阶段解耦**：第一阶段（切管分解，产出段字母表 Σ）和第二阶段（物料下料，追求利用率）各自独立求解；两者之间用**对偶价格 + 失败信息**做反馈回炉，迭代修正 Σ，直到第二阶段满意或达到迭代上限。

本质是 **Benders 式的启发式反馈**：第一阶段 = 主问题（决定 Σ），第二阶段 = 子问题（评估 Σ 的好坏并回传"割/信号"）。

---

## 1. 与路径乙的区别

| 维度 | 路径甲（本文档） | 路径乙（R&CG） |
|---|---|---|
| 两阶段关系 | 解耦，交替求解 | 合进单一 RSRMP |
| 段字母表 Σ | 第一阶段整批产出，反馈整批修正 | 作为主问题的行，单个动态生成 |
| 反馈机制 | 对偶价格 + 失败信息（启发式） | reduced-cost 定价（严格） |
| 收敛保证 | **无理论保证，可能震荡** | LP 层面严格收敛 |
| 实现复杂度 | 低（复用现有 solver） | 高（接近 branch-and-price） |
| 适用 | 快速上线 / 中小规模 | 追求最优 / 大规模 |

## 2. 两阶段定义

### 第一阶段：切管分解（产出 Σ）

**输入**：成品管需求（长度、根数）、每根管的禁焊区。
**输出**：段字母表 Σ = {段规格及其数量}，即中间订单。
**目标（你的修正版）**：
1. **焊口最少**（段尽量长/少）；
2. **花样最少**（段规格种类 |Σ| 尽量小，让不同管复用同一批段长）；
3. **禁焊区可行**（每根管的切点落在合法焊位）。

> 关键认知（已与你确认）：**不是单管贪心取最长**，而是"禁焊区约束下，为整个物料组联合选一个尽量小的段字母表 Σ"。①②天然冲突，需权衡。

建模：MILP。变量 = 每根管用哪种切法；约束 = 禁焊区合法性；目标 = 字典序（焊口数 → 段种类数）或加权。

### 第二阶段：物料下料（评估 Σ）

**输入**：Σ（第一阶段产出的段需求）、原料长度/库存。
**输出**：切割花样 + 使用次数；利用率。
**目标**：利用率最高（trim loss 最小）。
建模：标准一维下料，复用现有 `solver.py` 的 `_solve_group_scip_with_candidates`。

## 3. 反馈机制（核心，决定是否震荡）

这是你最担心的部分——"每次迭代怎么影响下次迭代，怎么保证不震荡"。设计三类反馈信号，**只增不减地累积约束**（保证单调、防震荡）：

### 信号 A：不可行段 → 禁止集（Forbidden Set）

- 若第二阶段发现某段规格 `k` 无论如何都凑不出高利用率的原料切法（例如 `ℓ_k` 与原料长度极不整除，单独切浪费巨大），把 `k` 标记进**禁止集 F**。
- 下一轮第一阶段求解时，**Σ 不得再产出 F 中的段规格**（硬约束）。
- **单调性**：F 只增不减 → 第一阶段可行域单调收缩 → 不会来回反复产出同一个坏段。

### 信号 B：对偶价格 → 段成本表（Cost Table）

- 第二阶段 LP 松弛的对偶价格 `π_k` 反映"每个段规格对利用率的边际贡献/代价"。
- 把 `π_k` 回传，更新第一阶段目标里的**段成本项**：`第一阶段目标 += Σ_k π_k · (产出的段k数量)`。
- 效果：第一阶段在"焊口/花样最少"之外，也开始**偏好第二阶段容易高利用率的段长**。
- **防震荡**：对 `π_k` 做**指数平滑**（`π_k ← α·π_k^new + (1-α)·π_k^old`, α≈0.5），避免单轮对偶剧烈波动导致 Σ 大幅跳变。

### 信号 C：整体质量 → Benders 式最优割（可选，进阶）

- 若把第二阶段利用率的下界作为第一阶段的一个代理目标，可加**最优割**：
  `η ≥ (第二阶段在当前Σ下的利用率损失) − Σ_k π_k·(Δ产出_k)`
  其中 η 是第一阶段引入的代理变量。
- 这让第一阶段"预估"改变 Σ 对第二阶段的影响，是最接近正统 Benders 的做法。工程上可先跳过，只用 A+B。

## 4. 主循环（伪代码）

```text
F = ∅                      # 禁止集
cost_table = 0             # 段成本表(初始无偏好)
best = None
for iter in 1..MaxIter:
    # ---- 第一阶段 ----
    Σ = 解切管分解MILP(需求, 禁焊区,
                       禁止集=F,
                       段成本=cost_table,
                       目标=字典序[焊口最少, 花样最少] + 段成本项)
    if Σ 无可行解:
        放宽F中最"冤枉"的约束 或 报告不可行; break

    # ---- 第二阶段 ----
    (花样, 利用率, π) = 解下料MILP(Σ, 原料)
    if 第二阶段不可行 (某些段凑不出):
        把责任段加入 F
        continue           # 直接回炉, 不更新best

    # ---- 记录 & 收敛判断 ----
    if 利用率 > best.利用率:
        best = (Σ, 花样, 利用率)
    if |利用率 - 上轮利用率| < ε 且 F 未变化 且 Σ 未变化:
        break              # 稳定, 收敛

    # ---- 反馈 ----
    cost_table = 指数平滑更新(cost_table, π, α=0.5)   # 信号B
    # 信号A已在第二阶段不可行时处理

return best
```

## 5. 防震荡的四条保证（直接回应你的顾虑）

1. **禁止集单调**（信号A）：F 只增不减，坏段一旦被禁不会复活 → 消除"这轮产出坏段、下轮又产出"的循环。
2. **对偶平滑**（信号B）：`π` 指数平滑，避免 Σ 因单轮对偶噪声大幅跳变。
3. **best 保留 + 早停**：记录历史最优，Σ 与 F 都稳定时立即停 → 不做无意义迭代。
4. **迭代上限 MaxIter 兜底**：即使不收敛，也在上限处返回历史最优 best，保证一定出解。

> 局限：以上是**启发式保证**，不是数学收敛证明。理论上仍可能停在非全局最优（第一/第二阶段各自最优 ≠ 全局最优）。若对最优性要求高，用路径乙。

## 6. 与现有 `solver.py` 的集成

| 现有组件 | 路径甲用法 |
|---|---|
| `_tiered_weld_candidates` / 切管逻辑 | 第一阶段 MILP 的花样候选 |
| `_generate_cut_candidates` | 第二阶段下料候选 |
| `_solve_group_scip_with_candidates` | 两阶段各自的 MILP 求解 |
| L1700+ 字典序目标 | 第一阶段目标（焊口→花样）；第二阶段目标（利用率） |
| `_solve_group_fallback` | 任一阶段失败时兜底 |

改动量：在现有单次求解外**包一层迭代循环 + 禁止集/成本表状态**，不需重写核心 MILP。这是路径甲的最大优势。

## 7. 何时选甲、何时选乙

- **选甲**：需要快速上线；问题规模中小；能接受"接近最优"；团队暂无 branch-and-price 经验。
- **选乙**：追求全局最优 / 大规模 / 段字母表高度不确定；有精力实现行列生成与整数化。
- **推荐路线**：先按 `route-b-rcg-design.md` 实现乙（用户已定）；甲作为落盘备选与降级方案，保留于此。
