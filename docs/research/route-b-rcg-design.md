## 0. 一句话概括

把"切管分解（第一阶段）"和"物料切割（第二阶段）"**合进一个受限主问题 RSRMP**，通过**同时生成列（切割花样）和行（中间订单 = 段规格）**来逼近全局最优。列生成负责补充更优花样，行生成负责补充更优的段规格。两者交替、由对偶价格驱动，收敛性最好、最不易在两阶段之间来回震荡。

---

## 1. 为什么是 R&CG，而不是简单反馈（路径甲）

我们的排料问题本质是**两阶段下料**：

- **第一阶段（切管 / splitting）**：把成品管按禁焊区切成"段"，段就是**中间订单（intermediate order）**。目标：焊口最少 + 段规格种类最少（花样最少）。
- **第二阶段（下料 / slitting）**：把原料按段需求切割拼焊成成品。目标：利用率最高。

难点：**中间订单（段规格集合 Σ）事先未知**。段规格既是第一阶段的产出，又是第二阶段的输入需求。

- 路径甲（解耦+反馈）：先猜一个 Σ，跑第二阶段，用对偶价格/失败信息回炉修正 Σ，反复迭代 → **可能震荡、收敛无理论保证**。
- 路径乙（R&CG）：把 Σ 当作**主问题的行**动态生成，用统一的对偶价格同时给两阶段的花样和新段规格定价 → **在 LP 松弛层面有严格的 reduced-cost 收敛判据**，是文献里对"中间订单未知"问题的标准解法（Column-Dependent-Rows, CDR 问题）。

> 文献结论（Grieco 2024 §3.2 / §4）：两阶段问题属于 **Column-Dependent-Rows** 类，行（中间订单）与列（花样）互相依赖，必须行列同时生成才能正确定价；仅列生成或仅反馈都不能保证收敛。

## 2. 主问题 RSRMP（Relaxed Short and Restricted Master Problem）

### 2.1 记号

| 符号 | 含义 |
|---|---|
| `Σ = {k}` | 中间订单（段规格）集合，即第一阶段产出、第二阶段消费的"段"。**动态生成（行）** |
| `ℓ_k` | 段规格 k 的长度 |
| `P1 = {p}` | 第一阶段切割花样（一根成品管 → 若干段的切法）。**动态生成（列）** |
| `P2 = {q}` | 第二阶段切割花样（一根原料 → 若干段的切法）。**动态生成（列）** |
| `a^1_{k,p}` | 花样 p 产出段 k 的数量（第一阶段，产出为正） |
| `a^2_{k,q}` | 花样 q 消费段 k 的数量（第二阶段，消费为负） |
| `d_j` | 成品管 j 的需求根数（第一阶段输入需求） |
| `x_p, y_q` | 花样 p、q 的使用次数（决策变量，LP 松弛下连续） |
| `π_k` | 段 k 的"守恒约束"对偶价格（联系两阶段的 linking dual） |
| `μ_j` | 成品管 j 需求约束的对偶价格 |
| `σ_ℓ` | 原料库存约束的对偶价格（若原料有限） |

### 2.2 约束结构（分块矩阵）

```
        列: P1(x)        P2(y)
行:  ┌─────────────┬─────────────┐
成品 │  A_demand    │      0      │  ≥ d      (第一阶段必须切出所有成品管)
需求 ├─────────────┼─────────────┤
段守 │  A11 (产出)  │  A22 (消费) │  = 0 / ≥0 (段: 第一阶段产出 = 第二阶段消费)
恒 k │   +a^1       │    -a^2     │           ← 这就是 linking constraints
     ├─────────────┼─────────────┤
原料 │     0        │   B_stock   │  ≤ S      (原料库存上限, 可选)
库存 └─────────────┴─────────────┘

min  Σ c_p x_p + Σ c_q y_q     (目标: 见 §4 多目标)
```

关键点：**"段守恒"这排约束的行数 = |Σ|**，而 Σ 是动态的。每新增一个段规格 k，就新增一行，同时必须新增至少一个产出它的 P1 列和一个消费它的 P2 列（否则该行恒为 0，无意义）——这就是"行列同时生成"。

> 对应 Grieco §3.2 式(7)(8)：第一阶段用 **set-partitioning（=）** 强制中间产物被全部消费（否则剩料浪费）；第二阶段用 **set-covering（≥）** 满足成品需求。我们的"成品管"在第一阶段是被分解的输入，所以我们的分块与其对称，方向相反，但结构同构。

### 2.3 初始化（Short 的含义）

- `A11` 初始化为单位阵：每个候选段规格配一个"只切一段"的平凡第一阶段花样。
- `A22` 初始化：对每个成品需求段，配一个整比可行（`ℓ_k / ℓ_成品` 合法）的平凡第二阶段花样。
- Σ 初始只放**最保守的一批段规格**（例如：每根成品管避开禁焊区后的"整根不切"分解，长度种类最少的那组）。

## 3. 三类定价子问题（Pricing Subproblems）

每轮解完 RSRMP 的 LP 松弛，拿到对偶 `(μ, π, σ)`，检查是否存在 reduced cost < 0 的新列/新行。**三类子问题**：

### 子问题 ①：第一阶段花样定价（新列 x_p）

给定当前段字母表 Σ 和对偶 π、μ，找一个新的"成品管切法"p，使其 reduced cost 最小：

```
rc(p) = c_p − μ_{j(p)} − Σ_k π_k · a^1_{k,p}
```

- `c_p`：该切法的成本（= 焊口数惩罚 + 段种类惩罚，见 §4）。
- 本质是一个**带禁焊区约束的整数背包 / 最短路**：在成品管 j 的长度上放置切点，切点必须避开禁焊区，切出的每段长度必须 ∈ Σ，目标最大化 `Σ π_k · a^1_{k,p} − c_p`。
- **禁焊区在这里生效**：切点位置合法性是子问题的硬约束。用 DP over 管长（位置离散化）实现：`f[pos] = max over 合法段长 s: f[pos - s] + π_s`，且 `pos - s → pos` 这一刀不落在禁焊区。

### 子问题 ②：第二阶段花样定价（新列 y_q）

给定 Σ 和对偶 π、σ，找一个新的"原料切法"q：

```
rc(q) = c_q + Σ_k π_k · a^2_{k,q} + σ_{stock}
```

- 本质是标准的**一维下料背包**：在原料长度内放置若干段（段长 ∈ Σ），最大化 `−Σ π_k · a^2_{k,q}`（因为消费 a^2 为正、π 为该段影子价），即"多用高影子价的段"。
- 这是经典 Gilmore-Gomory 列生成子问题，可用整数背包 DP 求解。

### 子问题 ③：中间订单生成（新行 k + 配套列）

当 ①② 都找不到改进列时，说明**当前段字母表 Σ 限制了解空间**，需要引入新段规格。这是 R&CG 相对纯列生成的核心增量：

- 识别"最亏"的第二阶段花样（trim loss 最大、影子价异常的段组合），推断一个能显著降低浪费的新段长 `ℓ_new`。
- 检查 `ℓ_new` 的**禁焊可行性**：是否存在成品管能在合法焊位切出该段（不能，则放弃）。
- 新增行 k=new，并**同步生成**：
  - 一个产出 `ℓ_new` 的第一阶段花样（否则新行永远是 0），
  - 一个消费 `ℓ_new` 的第二阶段花样。
- 用最大优先级插入定价树的最左分支，保证下一轮优先探索（Grieco §4.1）。

> **收敛判据**：当三类子问题全部找不到 reduced cost < 0 的候选时，RSRMP 的 LP 松弛达到全局最优。这是路径乙"不震荡"的理论根基——反馈不再是启发式回炉，而是对偶价格驱动的严格定价。

## 4. 多目标处理（对齐项目现状）

项目当前是**字典序多目标**（`solver.py` L1700+：先利用率、后焊口……）。R&CG 下的处理：

1. **主目标建进 `c_p / c_q`**：
   - 第二阶段成本 `c_q` = trim loss（利用率主目标）。
   - 第一阶段成本 `c_p` = `w_weld · 焊口数 + w_pattern · 段种类惩罚`。
2. **字典序用分层列生成实现**：先固定高优先级目标（如利用率）求 LP 最优，再在其最优面上加约束、优化次优先目标（焊口/花样）。或用足够大的权重差近似字典序（工程上更简单，先用这个）。
3. **花样最少化**：段种类数 = |Σ| 中被实际使用的规格数。在 `c_p` 里对"引入新段规格"加固定惩罚（setup cost），天然抑制字母表膨胀 → 直接实现你说的"禁焊区约束下花样最少"。

> 这一节直接回应你的修正：第一阶段目标 = **焊口最少 ⊕ 花样（段规格种类）最少**，二者都进 `c_p`，并由 π 对偶价格在全局层面权衡，而非单管贪心。

## 5. 主循环（伪代码）

```text
构造初始 RSRMP（Σ0 + 单位阵花样）
PatternGenPhase = True
while True:
    解 RSRMP 的 LP 松弛 → 得对偶 (μ, π, σ), 当前目标 obj
    improvement = 1 - obj / lastObj

    if PatternGenPhase:
        gen1 = 子问题①(π, μ)   # 第一阶段新花样
        gen2 = 子问题②(π, σ)   # 第二阶段新花样
        if gen1 or gen2 找到 rc<0:
            加入新列; 重置 aging 计数
        elif improvement 连续 K 轮 < ε:
            PatternGenPhase = False   # 转入行生成
    else:  # OrderGenPhase
        gen3 = 子问题③()      # 新中间订单(段规格) + 配套两列
        if gen3 找到能改进的新段:
            加入新行+新列; PatternGenPhase = True  # 回到列生成
        else:
            break             # 三类子问题都无改进 → LP 最优, 收敛

对 RSRMP 施加整数约束, 用 SCIP 求整数解（price-and-branch 或直接对最终列池求 MILP）
aging: 长期未使用的列定期淘汰, 控制问题规模
```

- **aging 机制**（Grieco §4）：每列带倒计时，最近未被用到就逐步淘汰，防止列池膨胀拖慢 LP。
- **整数化**：严格做法是 branch-and-price；工程折中是"生成足够列后对最终列池跑一次 MILP（SCIP）"，与现有 `solver.py` 的 `_solve_group_scip_with_candidates` 复用。

| 现有组件 | R&CG 复用方式 |

| `_generate_cut_candidates` | 退化为"初始列池"生成器；定价子问题②在其基础上做增量 |
| `_tiered_weld_candidates` | 第一阶段花样的初始池；定价子问题①替代其分层扩张 |
| `_solve_group_scip_with_candidates` | 作为最终列池的 MILP 整数化求解器 |
| `_solve_group_fallback` | R&CG 超时/失败时的兜底（保留） |
| L1700+ 字典序目标 | 映射到 §4 的 `c_p/c_q` 权重 + 分层列生成 |

**落地顺序建议**：
1. 先在**单物料组**上实现 RSRMP + 子问题①②（固定 Σ，纯列生成），验证能复现或优于现状。
2. 再加子问题③（行生成），打开段规格动态化。
3. 最后接 aging + price-and-branch 整数化。

## 8. 风险与开放问题

1. **整数化 gap**：LP 松弛最优 ≠ 整数最优。price-and-branch 复杂；先用"最终列池 MILP"折中，评估 gap 是否可接受。
2. **禁焊区离散化粒度**：子问题①的 DP over 位置，粒度太细则 DP 慢，太粗则漏解。需按项目真实数据（`真实数据字典-DGMOM批次.md`）标定。
3. **对偶退化 / 循环**：CSP 列生成常见对偶震荡。可加 dual-optimal inequalities（文献[1]）或稳定化（stabilization）。
4. **多目标字典序 vs 权重近似**：先用大权重差近似，若出现目标错位再上严格分层。
